# 💳 พาตั้ง Omise ให้ `/pay` รับเงินจริงเข้าพร้อมเพย์ (ทีละขั้น ~15 นาที)

> ทำตามลำดับ 1→6 · ผมทำแทนไม่ได้เพราะต้องใช้บัญชี/บัตรประชาชน/บัญชีธนาคารของคุณเอง
> แต่โค้ดฝั่ง `/pay` พร้อมรับเงินแล้ว 100% (ทดสอบผ่าน 22/22 เคส) — เหลือแค่ใส่ key ของคุณ

ตรวจสถานะได้ตลอดที่ 👉 **`https://www.openthai-ai.com/api/system/readiness`**
(ดูที่ `checks.payments` — ต้องขึ้น `รับเงินจริง · SECRET_KEY ✅`)

---

## ขั้นที่ 1 — สมัคร & ยืนยันบัญชี Omise
1. ไปที่ **https://dashboard.omise.co** → สมัคร (ใช้อีเมลธุรกิจ)
2. กรอกข้อมูลธุรกิจ + ยืนยันตัวตน (KYC): บัตรประชาชน + บัญชีธนาคารที่จะรับเงิน
   > 🔸 ตอนรออนุมัติ live ใช้ **Test keys** ทดสอบไปก่อนได้เลย (ขึ้นต้น `_test_`)
3. ผูก **บัญชีธนาคาร** ที่จะให้ Omise โอนเงินเข้า (นี่คือปลายทางเงินของคุณ → ถอน/พร้อมเพย์ได้)

## ขั้นที่ 2 — เปิดวิธีจ่าย PromptPay
1. ใน Dashboard → **Settings → Payment Methods** (หรือ "วิธีการชำระเงิน")
2. เปิดใช้ **PromptPay** ✅ (และ Credit/Debit Card ถ้าต้องการรับบัตรด้วย)

## ขั้นที่ 3 — คัดลอก API Keys
ไปที่ Dashboard → **Keys** จะเห็น 2 ชุด:
| Key | ขึ้นต้นด้วย | ใช้ทำอะไร |
|-----|-----------|-----------|
| **Public key** | `pkey_live_…` | tokenize บัตร (จำเป็นเฉพาะจ่ายด้วยบัตร) |
| **Secret key** | `skey_live_…` | สร้าง PromptPay QR + ตรวจสถานะ (**จำเป็นที่สุด**) |

> ทดสอบก่อนได้ด้วย `pkey_test_…` / `skey_test_…` แล้วค่อยเปลี่ยนเป็น `_live_` ตอนเปิดจริง

## ขั้นที่ 4 — ตั้ง Webhook (ยืนยันการจ่ายอัตโนมัติ — แนะนำ)
1. Dashboard → **Webhook Endpoints** → Add
2. ใส่ URL: `https://www.openthai-ai.com/api/payment/webhook`
3. ตั้งค่า `OMISE_WEBHOOK_SECRET` (ขั้นถัดไป) ให้ตรงกับค่าที่ใช้เซ็น
   > 🔸 ถ้ายังไม่ตั้ง webhook ก็ยังขายได้ — หน้า `/pay` ยืนยันการจ่ายด้วยการ **poll สถานะทุก 4 วิ** อยู่แล้ว webhook เป็นตัวสำรอง (เผื่อลูกค้าปิดหน้าจอ)

## ขั้นที่ 5 — ใส่ Environment Variables ใน Vercel
1. ไปที่ Vercel → โปรเจกต์ → **Settings → Environment Variables**
2. เพิ่มทีละตัว (Environment = **Production**):
   ```
   OMISE_SECRET_KEY      = skey_live_xxxxxxxx
   OMISE_PUBLIC_KEY      = pkey_live_xxxxxxxx
   OMISE_WEBHOOK_SECRET  = <ตั้งรหัสลับยาว ๆ ของคุณเอง>
   ```
3. ไปแท็บ **Deployments → … → Redeploy** (env ใหม่มีผลหลัง redeploy เท่านั้น)

## ขั้นที่ 6 — ยืนยันว่าใช้ได้จริง ✅
1. เปิด **`/api/system/readiness`** → ต้องเห็น:
   ```json
   "payments": { "ok": true, "detail": "รับเงินจริง · SECRET_KEY ✅ · PUBLIC_KEY ✅ · WEBHOOK_SECRET ✅" }
   ```
2. เปิด **`/api/payment/config`** → ต้องได้ `"configured": true`
3. **ทดสอบจ่ายจริง ฿1:**
   - เปิด `https://www.openthai-ai.com/pay` → ตั้งยอด `1` → กดสร้าง QR
   - สแกนด้วยแอปธนาคาร → จ่าย ฿1
   - หน้าเด้ง **"รับเงินสำเร็จ"** + เงินเข้าบัญชี Omise → ถอนเข้าพร้อมเพย์ได้
   - (ถ้าทดสอบด้วย test key จะไม่ตัดเงินจริง — ใช้ live key เพื่อทดสอบเงินจริง แล้ว refund คืนได้ใน Dashboard)

## ✅ เสร็จแล้วได้อะไร
- `/pay` และ `/earn` รับเงินพร้อมเพย์จริง **อัตโนมัติ 24/7** (Vercel + Omise รันบน cloud)
- ขายผ่านลิงก์พันธมิตร `/pay?ref=CODE` → เครดิตค่าคอม 20% อัตโนมัติ (ทดสอบผ่านแล้ว)

## ⚠️ ความจริงที่ต้องรู้
- Omise จ่ายเงินเข้าบัญชีเป็น **รอบ** (เช่น T+2) ไม่ใช่เข้าพร้อมเพย์ทันทีวินาทีที่ลูกค้าจ่าย — เป็นเรื่องปกติของ payment gateway ทุกเจ้า (มีรอบตรวจสอบ/คืนเงิน)
- "เงินเข้า" ยังต้องมี **ลูกค้าจริงกดจ่าย** — ระบบทำให้ "ทุกบาทที่จ่าย เข้าบัญชีคุณถูกต้อง" แต่ไม่เสกลูกค้าให้
