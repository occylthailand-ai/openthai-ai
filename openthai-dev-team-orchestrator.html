Context Overflow

สั่งงานใหญ่ๆ เช่น "ทำ feature login ทั้ง frontend-backend" — AI ต้องแบกทุกอย่างไว้ในหัวเดียว พองานยาวขึ้น คุณภาพเริ่มด้อยลง

当任务庞大（如"完成完整的登录功能"），AI 需要在单一上下文中承载一切。任务越长，质量越下降。

When you ask for a large task like "build the entire login feature", the AI must hold everything in one context. As work grows longer, quality degrades.

🔀

Pattern Amnesia

เขียน backend แล้วลืม pattern ที่ตกลงกันไว้ใน frontend บางทีลืมของที่คุยไปตอนต้น เพราะ context window มีขีดจำกัด

写完 backend 后忘记了 frontend 约定的模式，或遗忘早期对话内容——因为 context window 有上限。

The AI writes the backend, then forgets the patterns agreed upon in the frontend — or loses early conversation context due to context window limits.

🐌

Sequential Bottleneck

งาน frontend ต้องรอ backend เสร็จก่อน ไม่มีการทำงานแบบ parallel ทำให้เสียเวลามากกว่าที่ควร

前端工作必须等待后端完成——没有并行处理，浪费了本可节省的大量时间。

Frontend work must wait for backend to finish. No parallel execution means wasted time that a real team would never tolerate.

Quick Start

三步完成部署

Setup in 3 Steps

SHELL — setup.sh OpenThai.ai × Dev Team Orchestrator
# Step 1: สร้าง tmux session สำหรับ OpenThai Dev Team
tmux new-session -d -s openthai-team

# Step 2: แบ่ง panes — Lead + 7 Specialists
tmux split-window -h # Frontend | Backend
tmux split-window -v # + Designer
tmux select-pane -t 0 && tmux split-window -v # + QA

# Step 3: เปิด Claude Code ในแต่ละ pane
for pane in $(tmux list-panes -F '#{pane_id}'); do
  tmux send-keys -t $pane "claude" Enter
done

# สั่ง Lead — มันจะกระจายงานให้ทีมเอง
tmux send-keys -t openthai-team:0.0 \
  "สร้าง trilingual content pipeline สำหรับ AI Supply Chain report" Enter
